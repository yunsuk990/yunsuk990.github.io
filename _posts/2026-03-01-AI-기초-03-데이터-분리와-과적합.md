모델이 훈련 데이터에서는 잘 맞는데, 실제 데이터에서는 성능이 떨어지는 경우가 많습니다.
이 문제를 이해하려면 먼저 **데이터를 왜 나누는지** 알아야 합니다.

이번 글에서는 초보자 기준으로,
- 학습/검증/테스트 데이터의 차이
- 과적합(Overfitting)과 일반화(Generalization)
- 실무에서 자주 하는 실수
를 쉽게 정리해보겠습니다.

---

## 1) 데이터는 왜 나눌까?

모델 성능을 공정하게 확인하려면,
모델이 **처음 보는 데이터**에서 얼마나 잘 동작하는지 봐야 합니다.

같은 데이터로 학습도 하고 평가도 하면,
시험 문제와 답안을 미리 외운 뒤 시험 보는 것과 비슷합니다.
그래서 점수는 높게 나오지만 실제 실력은 아닐 수 있습니다.

---

## 2) 학습/검증/테스트 데이터의 역할

### ① 학습 데이터(Train)
모델이 규칙을 배우는 데이터입니다.
- 모델 파라미터를 직접 업데이트할 때 사용
- 가장 많은 비율을 차지

### ② 검증 데이터(Validation)
모델을 조정하는 데 쓰는 데이터입니다.
- 하이퍼파라미터(학습률, 트리 깊이 등) 튜닝
- 여러 모델 후보 중 어떤 모델이 좋은지 선택
- 조기 종료(Early Stopping) 판단

### ③ 테스트 데이터(Test)
최종 성적표 역할입니다.
- 모델 개발이 끝난 뒤 **한 번** 확인
- 모델의 실제 일반화 성능을 추정

핵심은 단순합니다.
- **Train:** 배우는 용도
- **Validation:** 고르는 용도
- **Test:** 최종 확인 용도

---

## 3) 과적합(Overfitting)이란?

과적합은 모델이 학습 데이터를 너무 외워서,
새 데이터에는 약해지는 현상입니다.

예를 들어,
- Train 정확도: 99%
- Validation 정확도: 78%

처럼 차이가 크면 과적합을 의심할 수 있습니다.

### 왜 생길까?
- 모델이 너무 복잡할 때
- 데이터가 적을 때
- 노이즈(잡음)까지 학습했을 때
- 학습을 너무 오래 했을 때

---

## 4) 일반화(Generalization)가 중요한 이유

실무에서는 “훈련 성능”보다
**새 데이터에서도 잘 작동하는 능력(일반화)** 이 더 중요합니다.

결국 우리가 원하는 것은
"이미 본 문제를 맞추는 모델"이 아니라
"처음 보는 문제도 잘 맞추는 모델"입니다.

---

## 5) 과적합 줄이는 대표 방법

1. **데이터 늘리기**
   - 가능한 한 다양한 데이터를 모으기
2. **정규화(Regularization)**
   - L1/L2, Dropout 등으로 과도한 학습 억제
3. **모델 단순화**
   - 너무 복잡한 모델 대신 적절한 복잡도 선택
4. **조기 종료(Early Stopping)**
   - 검증 성능이 나빠지기 시작하면 학습 중단
5. **교차검증(Cross Validation)**
   - 데이터 분할을 여러 번 바꿔 안정적으로 평가

---

## 6) 입문자가 자주 하는 실수

- 테스트 데이터를 개발 중에 계속 확인함
- 검증 데이터 없이 감으로 모델 선택
- 데이터 누수(Data Leakage)를 놓침
  - 예: 미래 정보를 과거 예측에 사용

테스트 데이터는 **마지막 한 번의 판정용**이라는 원칙만 지켜도
평가 신뢰도가 크게 올라갑니다.

---

## 정리

- 데이터 분리는 공정한 평가를 위한 기본 장치
- 과적합은 "외우기", 일반화는 "이해하기"
- 좋은 모델은 테스트 이전에 검증 단계에서 잘 관리됨

다음 글에서는 **정확도(Accuracy), 정밀도(Precision), 재현율(Recall), F1-score**를
입문자 관점에서 쉽게 정리해보겠습니다.
